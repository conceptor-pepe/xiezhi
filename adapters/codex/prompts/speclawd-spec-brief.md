# Speclawd Spec Brief

Prepare `spec-delta.md`, `design.md`, and `tasks.md` for the current change before implementation begins.

Use this order:

1. data structures
2. API contract
3. module coupling
4. task split

Use stable traceability IDs across artifacts:

- `REQ-*` for spec requirements
- `DES-*` for design decisions
- `TASK-*` for smallest shippable slices
